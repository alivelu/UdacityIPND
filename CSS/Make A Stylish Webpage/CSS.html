Understanding CSS

What is CSS?

CSS means cascading style sheet

What is Style?

Style means look or the way the website appears to the user

>What do sheets mean?

You can have several CSS files or sheets that will be referenced in HTML. And all of them will be used to find out the final and visible style to your page. With CSS the most specific rule is applied to every element.

What do cascading and sheets mean?

Cascading means you keep going down and down into the rules until you find the one that best describes the elements and then that rule gets applied.

In CSS you first write what's called a selector. And this defines what elements of the page a particular style applies to. You can select an HTML tag and that rule will apply to all tags on the page. Search and Replace

You can also think about CSS as a search and replace tool. You identify a class or a tag of the element you want to find (or match, in CSS terminology), and then what you want to do with it, or what property values to replace with different ones.

Order Matters

It also matters where you define the rules and in what order they are applied. Styles can be defined in different places and are applied in the following order, with definitions further down the list overwriting previous definitions:

  • the default style of a browser (different browsers have slightly different styles)
  • stylesheet in a separate file (this is what you will be mostly using)
  • stylesheet inside HTML (this can be done for small projects but is not ideal)
  • inline style in an element (this can also be done but should be avoided)
Specifics Matter

"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

The Box Revisited:

Everything on the webpage is a box.Good understanding of box model is important. There is the concept of margins, borders, padding, content all of these makes the box on the web page. Open the following webpage in Developer tools.click here. If we explore this box in developer tools, At the very center of the image we have Content. Content is the image or text that appears on the website In the next layer we have padding. This is the green color. The padding clears an area around the content. And it's effected by the back-ground color of the box. So think of it as the space inside of the box that protects the content directly Border then goes around the padding and the content. And the border is inherited from the color property of the box. And lastly the outer layer is the margin and this clears an area around the border. The margin doesn't have a background color and its completely transparent. And think of this as the space between boxes. Try to play along with the widths of each of these and see how it affects the box both here and in the computed tab. For example if you change the margin from 20 pixels to 30 pixels , we will notice that we have little extra space at the top. Navigate to the computed tab, we can notice that 30 pixels at top and 30 pixels at bottom. By default the size of the actual element is equal to the sum of the border, padding and content width. So if we wanted to compute the width of the element shown on the page. We would have left border of 10. Plus a left padding of 30, plus a content width of 200. Plus right padding of 30 and right border of 10. So our actual element width is 280 pixels. Not just the 200 pixel width of the content How much space the box will take on the screen without recalculating it each and every time you change one of these values? But luckily an easier model has been developed. we should plan to use that This is CSS style, box-sizing, border-box, this makes the calculation of element size include both the border and padding. So that it's easier to make layouts.
*{ box-sizing: border-box; }
You can just set the width of the overall box and no matter how you tweak... The padding in border sizes, the size of the box will always stay the same. Remember though that margin is not included in the size. However there is one drawback with this model. It's bean developed fairly recently so if you want to be sure that older browsers are also able to dispaly it, You need to add whats called browser specific prefixes. These prefixes are webkit, moz and ms You can just add them infront of style definition and they are defined like so.
*{
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
}

Box Sizing:
There are four main points that Jessica addressed about box sizing. HTML elements are boxes and each box has 4 components. Because there are so many components to each box, it can often be hard to get the size of a box just right. There are two techniques you can use to help deal with sizing issues. Set sizes in terms of percentages rather than pixels. Set the box-sizing attribute to border-box for every element. Image here Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.
Box Positioning:
Divs are block elements (as opposed to inline), so by default they take up the entire width of a page. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
